  - 基于血量、技能倾向、控制需求做最小可用敌方决策
  - 输出统一为 `PendingCommand`
  - 已支持按目标类型自动选取自身 / 友方 / 敌方 / 全体 / 随机目标
- `engine.ts` 已完成第二轮收缩：
  - 敌方命令生成已移出
  - 已拆出回合开始、命令执行、战斗结束、回合收尾等流程辅助函数
  - 当前更接近纯流程编排层
- `settlement.ts` 已补齐第一版完整结算流程：
  - 掉落生成
  - 升级检查与经验需求重算
  - 金币公式可配置化
  - 战斗归档追加写入
  - 区分胜利 / 失败 / 平局 / 逃跑的结算分支
- `BattleResultSummary` 字段说明与长期归档结构说明已补入 `第一版战斗系统字段边界说明.md`。
- 独立战斗测试页已建立，可脱离酒馆环境本地预览与调试。
- 已修复防御持续时机问题，防御减伤不再错误跨越预期结算边界。
- 已修复药剂缺省 `目标类型` 时被判定为“没有合法目标”的问题。
- 测试页提交行动后已自动推进到下一次玩家可操作时机或战斗结束，不再需要手动逐步点推进。
- 测试页已补充双方战斗属性与技能描述展示，基础读盘与对照信息已可直接查看。

### 需要推进

- 当前推荐顺序中的 UI 补强、`App.vue` 选择态下沉、`index.yaml` 按钮清单补充、正式敌方掉落表入口已完成。
- 当前底层优先级已切换为先完成 DND 六维化重构，先清掉 `精神` 与 `先攻修正` 的核心依赖，再继续速度与多动机制。
- `settlement.ts` 已支持优先读取 `规则配置.敌方掉落表`，现阶段剩余工作主要变为“继续内容化配置”，而非继续补结构。
- 当前 `engine.ts` 虽已显著收缩，但日志文本拼装与部分阶段细节仍在流程层；若后续继续增长，可再评估补 `log.ts` 或
  `schema_adapter.ts`。
- 当前测试页仍缺少检定/伤害过程拆解显示，无法直接对照
  `raw d20 / modifier / targetAC / base / reduced / shieldAbsorbed / final`。
- 当前缺少更清晰的调试信息视图，尚未把 `roll payload`、命令结构、效果列表做成稳定调试面板。
- 应尽快评估“技能库固定化 + MVU 最小字段化 + 脚本派生运行态”方案，避免技能定义、运行态和 AI 生成边界继续发散。

## 2. 后续目标边界

本计划继续聚焦“第一版战斗系统补全”，目标是：

1. 让当前战斗系统收敛为正式模块，而不是继续在 `engine.ts` 和 `App.vue` 中堆逻辑。
2. 固定引擎 / UI / 脚本入口 / 世界书的职责边界。
3. 补齐第一版用户体验缺口，但不引入第二版复杂系统。
4. 为后续多目标、多前排、AI 文本桥接保留结构空间。

以下内容仍不进入本计划：

- 网格地图战斗
- 复杂站位系统
- 完整反应链与插队技能
- 战斗动画演出系统
- 多前排复杂阵型
- 多楼层联动战斗实例管理

为避免边改边乱，后续建议按下面顺序推进：

1. 六维底层重构与样例数据迁移
2. 测试页与状态栏补六维观察入口
3. 速度决定出手顺序和次数的机制
4. 继续内容化 `敌方掉落表` 与奖励配置

按当前真实优先级，建议改为：

1. 六维底层重构完成后的回归验证与文档收口
2. 测试页补检定/伤害过程拆解
3. 速度机制接入与行动顺序扩展
4. 视增长情况补 `log.ts` / `schema_adapter.ts` / `compact_adapter.ts`

补充说明：

- 原顺序中的 `TargetPanel.vue`、`ItemPanel.vue`、`SkillPanel.vue` hover 技能摘要已完成，现已从推荐顺序中移除。
- 原顺序中的 `ai.ts`、`engine.ts` 第二轮收缩、`settlement.ts` 结算补完已完成，当前优先级已切换到 UI 补强与使用文档收尾。
- 原顺序中的 UI 补强、`App.vue` 选择态下沉、`index.yaml`
  说明收尾、正式掉落表入口也已完成，当前文档应视为“第一版主体完成，进入观察与内容扩展阶段”。
- 当前新增优先级的核心原因不是结构缺失，而是需要先收敛六维语义、运行态派生边界和调试可视化，否则后续继续加速度或内容会放大排错成本。

## 5. 验收标准

### 数据层验收

- `Schema.parse(initvar)` 通过
- 战斗前后 `Schema.parse(stat_data)` 仍通过

### 引擎层验收

- `engine.ts` 不再承担派生、命令校验、效果执行、结算回写等全部职责
- `derive / roll / effects / commands / settlement / ai` 分层明确
- 敌方决策输出统一为 `PendingCommand`，不再在 `engine.ts` 内联生成临时命令

### UI 层验收

- 玩家不需要手动改变量即可完整打完一场战斗
- 状态栏能展示回合、技能、目标、日志、胜负、结果
- 测试页能显示技能描述、战斗属性、关键结算路径
- 测试页能用于手动验证技能、道具、状态展示是否与运行时结果一致

### 世界书层验收

- AI 不再直接改写战斗运行时数值
- AI 只提交行动意图，脚本负责检定与结算

### 结算层验收

- 战斗结束后长期档案正确更新
- 掉落、经验、金币、归档都能正确生成
- `BattleResultSummary` 可直接供前端结果展示与归档摘要复用
