# 第一版战斗系统后续开发计划

> 目标：在当前多单位CTB战斗系统可运行的基础上，补完被动效果、控制状态、装备接入等核心缺口，收敛为可维护的正式方案。

---

## 1. 当前进度概览

### 已完成

- 数据层与 Schema 稳定化。
- 核心引擎模块拆分：`commands.ts` / `derive.ts` / `roll.ts` / `effects.ts` / `settlement.ts` / `ai.ts`。
- CTB（Count Time Battle）行动顺序系统：先攻高的单位重置值小，行动更频繁。
- 多单位战斗（2v3）：移除 `slice(0,1)` 限制，先攻队列支持任意数量单位。
- 敌方AI自动行动：评分选技能，同一次 `advanceBattle` 调用内完成，不需要额外 tick。
- 豁免成功半伤：`saving_throw` 成功时伤害减半，非伤害效果跳过。
- AoE技能正确命中全体：`all_enemies`/`all_allies` 忽略传入 `targetIds`。
- 目标选择器：`single_enemy` 显示敌方列表，`single_ally` 显示友方列表，AoE/self 不显示。
- 目标合法性兜底：传入不合法目标时自动 fallback 到自动选目标。
- 行动顺序条：测试页实时显示CTB队列，当前行动者高亮。
- 战斗脚本入口：`bridge.ts` / `store.ts` / `index.ts`。
- 完整结算流程：经验/金币/掉落/升级/归档。
- 状态栏UI：7个Vue组件，`useBattleCommand.ts` composable。
- 独立测试页：`界面/战斗测试/`，支持2v3样例数据。

### 已知缺口（按优先级）

| 优先级 | 模块 | 问题 |
|--------|------|------|
| 高 | 被动效果 | `runPassiveTrigger` 只处理 `restore_mp`，其他效果类型静默忽略 |
| 高 | 控制状态完整实现 | `silence` 未禁用技能，`fear`/`confuse`/`charm` 无实际效果 |
| 中 | 装备系统接入 | `装备栏` 字段存在但 `deriveUnitAttributes` 未读取装备加成 |
| 中 | 反应动作 | `标记.已用反应` 存在但引擎未使用 |
| 低 | 测试页调试视图 | 缺少检定/伤害过程拆解（raw d20 / modifier / base / shield / final） |
| 低 | 掉落内容化 | `敌方掉落表` 结构已有，但样例数据未配置具体掉落 |

## 3. 后续实施模块

### 3.1 被动效果扩展（当前优先级：高）

**目标**：`runPassiveTrigger` 支持所有效果类型，与 `applySkillEffects` 复用同一套执行逻辑。

需要推进：

1. `engine.ts` 的 `runPassiveTrigger` 改为调用 `applySkillEffects` 或提取公共执行函数
2. 支持触发时机：`battle_start` / `turn_start` / `turn_end`
3. 支持效果类型：`restore_mp`（已有）/ `heal` / `shield` / `apply_status` / `add_modifier` / `damage`
4. 样例数据补充至少一个非 `restore_mp` 的被动（如"回合开始恢复护盾"）

涉及文件：`engine.ts`、`effects.ts`、`initvar.yaml`

### 3.2 控制状态完整实现（当前优先级：高）

**目标**：所有10种状态产生实际游戏效果。

需要推进：

1. `silence`：`commands.ts` 校验时，若施法者有 silence 状态，禁止使用 MP > 0 的技能
2. `fear`：AI决策时降低攻击倾向，或有概率跳过行动
3. `confuse`：有概率攻击己方随机目标
4. `charm`：强制攻击己方血量最高目标
5. `taunt`：`commands.ts` / AI 强制选择嘲讽来源为目标

涉及文件：`commands.ts`、`ai.ts`、`effects.ts`

### 3.3 装备系统接入（当前优先级：中）

**目标**：`deriveUnitAttributes` 读取 `装备栏` 数据，武器/防具属性加成生效。

需要推进：

1. Schema 确认装备定义表结构
2. `derive.ts` 在属性派生时叠加装备加成
3. 样例数据补充装备定义

涉及文件：`derive.ts`、`schema.ts`、`initvar.yaml`

### 3.4 测试页调试视图（当前优先级：低）

**目标**：测试页能展示检定/伤害过程拆解，便于排错。

需要推进：

1. `roll payload` 展示：raw d20、modifier、目标AC、命中结果
2. 伤害拆解：base、defense减免、护盾吸收、最终入账
3. 命令结构展示：当前待处理指令的完整内容

涉及文件：`界面/战斗测试/App.vue`

### 3.5 掉落内容化（当前优先级：低）

**目标**：为样例敌人配置具体掉落表。

需要推进：

1. `initvar.yaml` 的 `敌方掉落表` 补充哥布林系列掉落配置
2. 验证结算后背包正确更新

涉及文件：`initvar.yaml`、`settlement.ts`

## 4. 推荐后续顺序

1. **被动效果扩展** — 影响所有被动技能，优先级最高
2. **控制状态完整实现** — silence/charm/confuse 直接影响战斗深度
3. **装备系统接入** — 属性派生补完
4. **测试页调试视图** — 排错工具
5. **掉落内容化** — 纯内容配置，随时可做

## 5. 验收标准

### 引擎层
- 被动效果所有类型均可触发，日志可见
- 控制状态对行动产生实际限制
- 装备加成正确叠加到战斗属性

### UI层
- 测试页能完整打完2v3战斗并正确结算
- 行动顺序条实时反映CTB状态
- 目标选择器按技能类型正确显示

### 数据层
- `Schema.parse(initvar)` 通过
- 战斗前后 `Schema.parse(stat_data)` 仍通过

---

## 6. 工程验证

每次关键改动后：

1. `pnpm dev` 启动测试页
2. 手动打完一场2v3战斗，确认所有单位正常轮流行动
3. 验证修改功能的日志输出是否符合预期
4. 确认结算结果正确生成
