# wangyou 当前战斗系统：检定与伤害计算说明

本文对应当前代码实现（`src/wangyou/脚本/战斗系统`）。

---

## 1. 当前回合流程（多单位CTB）

当前支持多单位战斗（2v3等），使用CTB（Count Time Battle）行动顺序：

- 每个单位有 `行动计数器`，初始值 = `max(10, 100 - 先攻×3) + random(0, base×0.5)`
- 计数器最小的单位先行动
- 行动后重置为 `当前最小计数器 + max(10, 100 - 先攻×3)`
- 先攻高的单位重置值小，行动更频繁

流程顺序：

1. `turn_start`：`getNextActorByCTB` 选出计数器最小的存活单位
2. `select_action`：玩家输入或AI生成指令
3. `resolve_action`：执行技能/道具/防御/逃跑
4. 检查胜负：胜负/逃跑 → `battle_end`；否则 `finishRound`
5. `finishRound`：重置行动者计数器，`tickStatus`，冷却-1，清理防御中

敌方回合在同一次 `advanceBattle` 调用内完成，不需要额外tick。

## 2. 属性派生公式（运行时）

代码在 `derive.ts`：

- `生命层次 = 1 + floor((等级 - 1) / 4)`
- 每提升1个生命层次，`力量/敏捷/体质/智力/感知/魅力` 各+1，`幸运` 不增加
- `物理防御 = 10 + 体质 + floor(等级 / 2)`
- `精神防御 = 10 + 感知 + floor(等级 / 2)`
- `命中加值 = floor(敏捷 / 3)`
- `闪避加值 = floor(敏捷 / 3)`
- `先攻 = 敏捷 + floor(幸运 / 3)`
- `治疗强度 = floor(感知 / 2)`
- `Shield` 初始为 0

## 3. 命中/豁免/逃跑检定公式

### 3.1 attack_roll（攻击检定）

代码在 `roll.ts → resolveAttackCheck`：

- `modifier = 攻击属性 + 技能命中加值 + 行动者.命中加值 + 幸运微调`
- `finalRoll = d20 + modifier`（有优势/劣势时变2d20取高/低）
- 命中条件：自然20必中；自然1必失；否则 `finalRoll >= 对抗防御值`

### 3.2 saving_throw（目标豁免）

代码在 `roll.ts → resolveSavingThrow`：

- 目标掷骰：`d20 + 目标豁免属性 + 幸运微调`
- `DC = 技能基础DC`（未配置时用 `10 + 攻击属性`），叠加生命层次控制差修正
- 豁免成功：自然20必成；自然1必失；否则 `d20 + modifier >= DC`
- **豁免成功时：伤害减半，非伤害效果（状态附加等）跳过**

### 3.3 escape（逃跑）

- `modifier = 敏捷 + 感知 + 幸运微调`
- `finalRoll = d20 + modifier >= 逃跑基础DC（默认13）`

## 4. 伤害/治疗/护盾计算

### 4.1 伤害公式

代码在 `effects.ts → calcDamage`：

```
base = 行动者[scale属性] × ratio + flat
defense = fire/arcane → 精神防御；其他 → 物理防御
defendingFactor = 防御中 ? 0.6 : 1.0
tierFactor = max(0.4, 1 + (攻击者生命层次 - 防御者生命层次) × 0.1)
damage = max(1, floor((base - defense × 0.35) × defendingFactor × tierFactor))
```

入账顺序：先扣 `Shield`，剩余再扣 `HP`。

### 4.2 治疗公式

```
heal = max(1, floor(行动者[scale属性] × ratio + flat + 行动者.治疗强度))
```

### 4.3 护盾

```
amount = max(1, floor(行动者[scale属性] × ratio + flat))
```

### 4.4 道具

- `回复HP/MP`：直接加，受最大值钳制
- 满值时恢复0，但数量仍消耗

## 5. 状态与控制效果

| 状态 | 效果 |
|------|------|
| burn/poison/bleed | `tickStatus` 时造成DOT，伤害 = `max(1, floor(强度 ?? 4))` |
| stun/freeze | `是否可行动 = false` |
| silence | 禁止使用 MP>0 的技能（`commands.ts` 校验） |
| taunt | 强制攻击嘲讽来源（`commands.ts` + `ai.ts`） |
| fear | AI 30%概率改为防御 |
| confuse | AI 50%概率攻击己方随机目标 |
| charm | AI 强制攻击己方血量最高目标 |

状态和修正器每次 `tickStatus` 时 `剩余回合 - 1`，降到0移除。

## 6. 被动效果

代码在 `engine.ts → runPassiveTrigger`，调用 `effects.ts → applyEffectList`：

- 支持触发时机：`battle_start` / `turn_start` / `turn_end`
- 支持所有效果类型：`restore_mp` / `heal` / `shield` / `apply_status` / `add_modifier` / `damage`
- 被动以自身为 actor 和 target（`self` 语义）
