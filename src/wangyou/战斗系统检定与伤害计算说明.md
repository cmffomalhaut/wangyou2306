# wangyou 当前战斗系统：检定与伤害计算说明

本文对应当前代码实现（`src/wangyou/脚本/战斗系统`），用于解释：

- 战斗一回合是怎么推进的
- 日志里的“检定结果=23”是怎么算出来的
- 最终扣血是怎么从技能数值算出来的
- 道具、防御、状态、逃跑的判定逻辑

---

## 1. 当前回合流程（1v1）

当前初始化是 1v1（各阵营只取 1 名单位）：

- 见 `initializeBattleState`：`ally` 与 `enemy` 都是 `slice(0, 1)`。
- 每次调用 `advanceBattle` 只推进状态机的一步。
- 在测试页中，玩家提交行动后已改为自动连跑到“下一次玩家可操作”或“战斗结束”。

流程顺序（简化）：

1. `turn_start`：确定当前行动者（按先攻序列）。
2. `select_action`：玩家/AI 选择行动。
3. `resolve_action`：执行技能/道具/防御/逃跑。
4. 行动后检查胜负：
   - 若分出胜负或逃跑成功 -> `battle_end`
   - 否则进入下一个行动者
5. 当先攻队列走完一圈时触发 `finishTurn`：
   - 状态回合数-1、修正器回合数-1
   - 技能冷却-1
   - 清理 `防御中`
   - 处理 `turn_end` 被动

## 2. 属性派生公式（运行时）

角色档案里的七维会派生为战斗属性，代码在 `derive.ts`：

- `生命层次 = 1 + floor((等级 - 1) / 4)`
- 每提升 1 个生命层次，`力量/敏捷/体质/智力/感知/魅力` 运行时各 `+1`，`幸运` 不增加
- `物理防御 = 10 + 体质 + floor(等级 / 2)`
- `精神防御 = 10 + 感知 + floor(等级 / 2)`
- `命中加值 = floor(敏捷 / 3)`
- `闪避加值 = floor(敏捷 / 3)`
- `先攻 = 敏捷`
- `治疗强度 = floor(感知 / 2)`
- `Shield` 初始为 0

## 3. 命中/豁免/逃跑检定公式

### 3.1 attack_roll（攻击检定）

代码在 `roll.ts -> resolveAttackCheck`：

- `modifier = 攻击属性 + 技能命中加值 + 行动者.命中加值 + 幸运微调`
- `finalRoll = d20 + modifier`（有优势/劣势时变 2d20 取高/低）
- 命中条件：
  - 自然20必定成功
  - 自然1必定失败
  - 否则 `finalRoll >= 对抗防御值`

`对抗防御值`取决于技能配置：

- `护甲等级` 或 `物理防御` 或 `精神防御`

### 3.2 saving_throw（目标豁免）

代码在 `roll.ts -> resolveSavingThrow`：

- 目标掷骰：`d20 + 目标豁免属性 + 幸运微调`
- `DC = 技能基础DC`（若未配置，使用 `10 + 攻击属性`），再叠加生命层次控制差修正
- 豁免成功条件：
  - 自然20必成功
  - 自然1必失败
  - 否则 `d20 + modifier >= DC`

注意：在技能执行里，`saving_throw` 的 **技能是否生效** 使用 `checkPassed = !roll.success`。

### 3.3 escape（逃跑）

代码在 `roll.ts -> resolveEscapeAction`：

- `modifier = 敏捷 + 感知 + 幸运微调`
- `finalRoll = d20 + modifier`
- 成功条件：自然20成功；自然1失败；否则 `finalRoll >= 逃跑基础DC`

默认 `逃跑基础DC = 13`（来自 `规则配置.资源规则`）。

## 4. 伤害/治疗/护盾/道具计算

### 4.1 伤害公式

代码在 `effects.ts -> calcDamage`：

- `base = 行动者[scale属性] * ratio + flat`
- 防御项：
  - 若 `damageType` 是 `fire` 或 `arcane` -> 用目标 `精神防御`
  - 否则 -> 用目标 `物理防御`
- 防御动作减伤：
  - 目标 `防御中 = true` 时乘 `0.6`
  - 否则乘 `1`
- 最终：
  - `damage = floor((base - defense * 0.35) * defendingFactor * 生命层次乘区)`
  - 最低伤害为 1

其中：

- `生命层次乘区 = 1 + (攻击者生命层次 - 防御者生命层次) * 生命层次伤害修正`
- 当前默认 `生命层次伤害修正 = 0.1`

入账顺序：先扣 `Shield`，剩余再扣 `HP`。

### 4.2 治疗公式

代码在 `effects.ts -> calcHeal`：

- `healRaw = 行动者[scale属性] * ratio + flat + 行动者.治疗强度`
- `heal = floor(healRaw)`，最低为 1
- 最终回复值受 `HPMax` 上限钳制

### 4.3 技能恢复 MP

`restore_mp` 效果：

- `amount = floor(flat + ratio * 行动者.智力)`（最低 1）
- 最终回复值受 `MPMax` 上限钳制

### 4.4 技能护盾

`shield` 效果：

- `amount = floor(行动者[scale属性] * ratio + flat)`（最低 1）
- 直接加到目标 `Shield`

### 4.5 道具（药水）

代码在 `engine.ts -> resolveItemAction`：

- `回复HP`: `HP = clamp(HP + 回复HP, 0, HPMax)`
- `回复MP`: `MP = clamp(MP + 回复MP, 0, MPMax)`
- 满值时会出现“恢复 0”，但道具数量仍会消耗 1

目标类型说明：

- 若道具未写 `目标类型`，当前逻辑默认按 `self` 处理（已修复）。

## 5. 状态与回合结算

状态应用与结算代码在 `effects.ts` 与 `engine.ts`：

- `burn/poison/bleed`：在 `finishTurn -> tickStatus` 时造成 DOT
  - DOT 伤害 = `max(1, floor(status.强度 ?? 4))`
- `stun/freeze`：使单位 `是否可行动 = false`
- 状态和修正器每整轮（先攻队列走完一圈）`剩余回合 - 1`
- 剩余回合降到 0 则移除

## 6. 为什么会出现“检定23，但扣血看不懂”

以默认数据里的 `火焰箭 fire_bolt` 为例（`initvar.yaml`）：

- 我方伊芙琳属性：
  - 智力 = 10
  - 敏捷 = 8
  - 幸运 = 5，因此幸运微调 = 0
  - 命中加值 = floor(敏捷8/3) = 2
- 火焰箭检定配置：
  - 攻击属性 = 智力
  - 命中加值 = +2
  - 对抗防御 = 精神防御

所以：

  - `modifier = 10 + 2 + 2 + 0 = 14`
- 如果本次 d20 原始点数是 4
- `finalRoll = 4 + 17 = 21`

日志会显示“检定结果为 21”。

然后算伤害：

- 火焰箭伤害参数：`scale=智力, ratio=1.4, flat=8, damageType=fire`
- `base = 10*1.4 + 8 = 22`
- 目标哥布林 `精神防御 = 10 + 感知5 + floor(等级2/2) = 16`
- 非防御中：`defendingFactor = 1`
- 双方生命层次同为 1，因此层次乘区为 1
- `damage = floor((22 - 16*0.35)*1*1) = floor(16.4) = 16`

若目标当时有护盾，例如 `Shield=6`：

- 先扣护盾 6
- 再扣 HP 12

## 7. 当前实现与直觉不一致的点（已知）

- 日志里只显示了“检定结果”和“受伤数值”，未直接打印完整公式分解。
- `闪避加值`、`异常抗性`、`控制强度`、`抗性`字段目前未真正参与伤害/命中计算。
- 防御减伤系数在代码中写死 `0.6`（即减伤 40%），与配置项数值是同一个语义，但当前不是动态读取配置。
- 目前是 1v1 初始化，不是多单位混战。
