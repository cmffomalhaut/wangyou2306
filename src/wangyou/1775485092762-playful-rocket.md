# 战斗系统六维化与底层重构计划

## 修改任务登记

- 修改目标：将 `src/wangyou` 战斗系统从现有五维体系重构为 DND 风格六维体系（力量、敏捷、体质、智力、感知、魅力），同步重做核心派生公式与相关底层字段，使系统尽量依赖少量核心数值，复杂性主要由派生关系与掷骰波动提供。
- 问题现象：当前系统使用 `五维 = 力量/魅力/体质/智力/精神`，其中 `精神` 混合了承担法术、感知、意志、治疗等多个职责，`先攻修正` 又作为额外基线项补丁式存在，导致 DND 语义不清晰、速度与感知/反应边界混乱、后续扩展 3v3 与更标准规则时会越来越难维护。
- 预期结果：完成六维字段替换、运行时属性重派生、检定/伤害/治疗/逃跑/状态相关公式收敛，并给出一套“少核心参数、强派生、运气有影响但不过度支配”的第一版底层方案。
- 非目标：本次不直接实现完整职业系统、装备词条、站位距离、借机攻击、反应链、法术位、豁免熟练体系，也不在本次计划中同时落地完整 3v3 和速度多动机制实现。
- 涉及文件：`src/wangyou/schema.ts`、`src/wangyou/世界书/变量/initvar.yaml`、`src/wangyou/脚本/战斗系统/derive.ts`、`src/wangyou/脚本/战斗系统/roll.ts`、`src/wangyou/脚本/战斗系统/effects.ts`、`src/wangyou/脚本/战斗系统/types.ts`、`src/wangyou/界面/战斗测试/App.vue`、`src/wangyou/界面/状态栏/components/BattleUnitCard.vue`、`src/wangyou/战斗系统检定与伤害计算说明.md`、`src/wangyou/第一版战斗系统字段边界说明.md`、`src/wangyou/第一版战斗系统后续开发计划.md`、`src/wangyou/DND通用战斗系统具体脚本设计.md`
- 影响范围：长期字段、运行时派生、检定公式、伤害与治疗公式、UI 展示、样例数据、说明文档。
- 风险点：高风险；涉及公式、默认值、长期字段与运行态桥接、测试页调试信息、未来速度机制入口。
- 复现方式：查看当前实现中 `精神` 同时控制精神防御、治疗强度、部分恢复、豁免属性等；`先攻` 由 `力量 + 魅力 + 先攻修正` 决定，不符合六维直觉。
- 验证方式：完成六维替换后，使用测试页核对单位六维、派生属性、攻击/豁免/伤害/治疗/逃跑日志，并确认样例战斗仍可从开始到结算闭环运行。

## 现状诊断

### 当前长期字段问题

- `src/wangyou/schema.ts:421` 当前角色长期基础字段为 `五维`，内容是：`力量 / 魅力 / 体质 / 智力 / 精神`。
- 这套结构不是 DND 六维；缺少 `敏捷` 与 `感知`，`精神` 同时承担了意志、防法、治疗、法力恢复等多重职责。
- `src/wangyou/schema.ts:440` 当前 `派生基线` 额外包含 `先攻修正`，说明先攻规则本身无法仅由主属性自然推出，只能靠补丁值修正。

### 当前运行时派生问题

- `src/wangyou/脚本/战斗系统/derive.ts:10`：`先攻 = 派生基线.先攻修正 + floor((力量 + 魅力) / 4)`
- `src/wangyou/脚本/战斗系统/derive.ts:11`：`物理防御 = 10 + 体质 + floor(等级 / 2)`
- `src/wangyou/脚本/战斗系统/derive.ts:12`：`精神防御 = 10 + 精神 + floor(等级 / 2)`
- `src/wangyou/脚本/战斗系统/derive.ts:13`：`命中加值 = floor((力量 + 智力) / 4)`
- `src/wangyou/脚本/战斗系统/derive.ts:14`：`闪避加值 = floor(魅力 / 5)`
- `src/wangyou/脚本/战斗系统/derive.ts:15`：`治疗强度 = floor(精神 / 3)`

这些问题集中表现为：

- `魅力` 被拿去参与先攻和闪避，不符合常见 TRPG 直觉。
- `精神` 既像感知又像意志又像法术属性，边界失真。
- `命中加值` 混合力量与智力，导致近战/远程/法术缺少明确分层。
- 等级直接进入防御，但并未形成成体系的熟练/成长逻辑，结果是“等级影响一点，但规则理念并不统一”。

### 当前检定问题

- `src/wangyou/脚本/战斗系统/roll.ts:46` 攻击检定加值 = `攻击属性 + 技能命中加值 + 命中加值`
- `src/wangyou/脚本/战斗系统/roll.ts:60` 豁免检定 = 目标 `豁免属性`
- `src/wangyou/脚本/战斗系统/roll.ts:62` 豁免 DC = `基础DC` 或 `10 + 攻击属性`
- `src/wangyou/脚本/战斗系统/roll.ts:79` 逃跑检定 = `先攻 + 魅力`

问题在于：

- 检定体系目前没有把“命中”“豁免”“速度”“观察”“意志”分成清晰来源。
- 若后续改为六维，`敏捷` 与 `感知` 必须进入至少一部分核心检定，不然六维只是改名，没有真重构。

### 当前效果问题

- `src/wangyou/脚本/战斗系统/effects.ts:205` 的 `restore_mp` 直接用 `精神`
- `src/wangyou/脚本/战斗系统/effects.ts:153` 的治疗公式附加 `治疗强度`
- `src/wangyou/脚本/战斗系统/effects.ts:91-97` 的 modifier 映射也仍是五维语义

这意味着只要改六维，`effects.ts` 与技能配置中的 `scale` 字段也必须整体校正。

## 设计目标

### 核心原则

本次底层重构建议遵循以下原则：

1. 长期主属性尽量少，只保留六维本体，不再继续堆很多“半主属性”。
2. 大多数战斗表现来自“六维 -> 派生属性 -> 检定结果 -> 效果结算”。
3. 掷骰要有波动，但不能压过构筑与属性差；即“运气影响结果，但不主导体系”。
4. 规则应支持未来最多 3v3，而不是只为当前 1v1 临时打补丁。
5. 第一版不追求纯正 DND 5e 复刻，而是做“DND 六维语义 + 适合当前项目的数据收敛版本”。

### 少核心数值的建议解释

“尽量少的核心数值”这里建议理解为：

- 长期层只保留：六维、等级、HP/MP 基础资源、极少数派生基线
- 运行时核心判定只围绕：命中、防御、豁免、速度、治疗、资源效率
- 不再引入大量二级基础面板，如暴击率、穿甲、格挡率、韧性、精神穿透等
- 复杂感来自：
  - 技能目标与效果设计
  - 六维到派生的差异
  - 检定类型选择
  - 状态和回合节奏
  - d20 与伤害浮动

## 推荐底层方案

### 一、长期主属性改为六维

把 `角色档案.五维` 升级为 `角色档案.六维`，字段固定为：

- `力量`
- `敏捷`
- `体质`
- `智力`
- `感知`
- `魅力`

推荐取舍：

- 不保留 `精神` 作为长期主属性。
- 不把 `感知` 和 `意志` 再拆成两个基础属性。
- `精神防御` 作为运行时派生保留，但其来源从 `精神` 改为 `感知` 或 `感知 + 魅力` 的组合方案。

### 二、派生基线做减法

当前 `派生基线` 建议收敛为极少数“模板修正项”，而不是拿来补主规则漏洞。

推荐保留：

- `护甲等级` 或将其改名为 `护甲基线`
- `控制强度`
- `异常抗性`

推荐移除或逐步废弃：

- `先攻修正`

原因：

- 一旦有 `敏捷`，先攻/速度就应该主要从 `敏捷` 派生。
- 若继续保留 `先攻修正`，就会再次变成“规则没说清楚，只能靠补丁校平”的状态。
- 如确实需要职业差异，可后续引入更明确的 `速度修正` 或职业模板，而不是沿用模糊的 `先攻修正`。

### 三、派生体系重定

推荐第一版派生方向如下：

- `物理防御 = 10 + 体质 + floor(等级 / 2)`
- `精神防御 = 10 + 感知 + floor(等级 / 2)`
- `速度/先攻 = 敏捷 + floor(感知 / 2)` 或更保守的 `敏捷 + floor((感知 - 10) / 2)` 的项目化版本
- `命中加值` 不再由固定双属性混合统一生成，而是保留“技能决定攻击属性”的主线
- `闪避加值 = floor(敏捷 / 3)` 或与速度共享来源
- `治疗强度 = floor((感知 + 魅力) / 4)` 或 `floor(感知 / 2)`
- `MP 恢复/法术强度` 优先由 `智力` 驱动，避免继续把“精神”残影留在系统里

这里的关键取舍是：

- `敏捷` 管速度、闪避、部分命中
- `感知` 管精神防御、感知类豁免、治疗/支援稳定性
- `智力` 管法术命中/DC/资源效率
- `魅力` 主要保留给社交、心智类效果、部分支援系技能，而不是继续承担先攻底层职责

### 四、检定体系收敛

推荐继续使用现有三类大框架：

- `attack_roll`
- `saving_throw`
- `escape`

但底层解释改成六维语义：

- `attack_roll`：由技能声明攻击属性，常见为 `力量 / 敏捷 / 智力 / 感知 / 魅力`
- `saving_throw`：由技能声明目标使用哪一维来抗性，常见为 `体质 / 敏捷 / 感知`
- `escape`：主改为 `敏捷 + 感知` 或 `敏捷 + 魅力`；推荐前者，因逃跑更像反应与观察机会

为了满足“运气有影响但不能太大”，建议：

- 保持 `d20`
- 但逐步降低“属性再次叠加两层”的问题，避免一个检定同时吃 `攻击属性 + 命中加值 + 复合命中加值` 过度膨胀
- 推荐把 `命中加值` 收敛为小幅派生项，或直接取消统一 `命中加值`，改成技能命中修正 + 主属性 + 少量职业/状态修正

### 五、伤害与治疗体系收敛

推荐方向：

- 伤害仍保留 `scale + ratio + flat`，因为这已足够灵活
- 但限制 `scale` 只从六维中取值
- 物理系技能多用 `力量/敏捷`
- 法术系技能多用 `智力/感知/魅力`
- 治疗默认使用 `感知` 或 `感知 + 魅力`
- `restore_mp` 默认改由 `智力` 或 `感知` 驱动，不再使用旧 `精神`

这能保证系统复杂度更多来自：

- 技能搭配
- 属性倾向
- 检定是否命中/豁免成功
- 防御和护盾入账路径

而不是来自一大堆额外二级数值。

## 行为变更记录

### 变更点：长期主属性从五维改为六维

- 原因：当前五维不符合 DND 语义，`精神` 职责过载。
- 旧行为：长期角色使用 `力量/魅力/体质/智力/精神`。
- 新行为：长期角色使用 `力量/敏捷/体质/智力/感知/魅力`。
- 为什么不用其他方案：继续保留 `精神` 并打补丁，会让未来速度、感知、意志、法术定位一直混乱。
- 影响范围：`schema.ts`、`initvar.yaml`、`derive.ts`、`roll.ts`、`effects.ts`、UI、说明文档。

### 变更点：先攻修正从核心依赖中移除

- 原因：有了 `敏捷` 后，速度应主要来自属性逻辑，而不是额外补丁字段。
- 旧行为：`先攻 = 先攻修正 + floor((力量 + 魅力) / 4)`。
- 新行为：`速度/先攻` 主要由 `敏捷` 与少量辅助属性派生，`先攻修正` 退场或降级为可选模板修正。
- 为什么不用其他方案：继续依赖 `先攻修正` 会让六维化流于表面。
- 影响范围：`derive.ts`、`roll.ts`、未来速度机制设计、说明文档。

### 变更点：精神相关公式拆解到智力/感知/魅力

- 原因：`精神` 旧字段同时承担法术、心智、防御、治疗职责，耦合过高。
- 旧行为：精神防御、治疗强度、部分回蓝、豁免依赖都可能落在 `精神`。
- 新行为：
  - 法术/资源效率更多交给 `智力`
  - 精神防御与感知类抗性交给 `感知`
  - 部分支援/心智类效果可使用 `魅力`
- 为什么不用其他方案：继续复用单一 `精神` 只会把六维重新压扁成伪五维。
- 影响范围：`derive.ts`、`roll.ts`、`effects.ts`、技能配置与说明文档。

- 场景：六维重构后的默认样例战斗
- 前置条件：默认角色与技能已迁移到六维字段

## 风险与排错顺序

### 重点风险

- 字段迁移风险：`schema.ts` 与 `initvar.yaml` 不同步会直接导致 parse 失败。
- 枚举遗漏风险：技能 `scale`、`攻击属性`、`豁免属性`、modifier 映射若残留 `精神` 会造成运行期错误。
- 公式错位风险：UI 展示、日志说明和真实实现若不同步，会让排错极其困难。
- 平衡风险：六维重构后默认样例数值可能会失衡，需要至少做一次手动样例校平。

### 推荐排错顺序

1. 先验证 `Schema.parse(initvar)` 能通过
2. 再验证 `derive.ts` 是否只读取六维，不再读旧 `精神`
3. 再验证 `roll.ts` 与 `effects.ts` 是否无旧字段残留
4. 再验证测试页是否全部改成六维显示
5. 最后手动打完一场战斗，核对日志与结算

---

## 与后续速度机制的关系

- 六维重构应先于速度机制实现。
- 原因：如果先做速度多动，再把五维改六维，速度来源会被改第二次，等于把同一高风险逻辑重复返工。
- 六维化完成后，速度机制就可以自然建立在 `敏捷` 为主、`感知` 为辅的基础上。
- 因此后续建议顺序改为：
  1. 六维底层重构
  2. 样例数据与测试页稳定
  3. 再做速度决定出手顺序和次数

## 默认取舍

- 默认不追求完全照搬 DND 5e 规则书。
- 默认保留当前统一命令模型 `skill/item/defend/escape`。
- 默认等级仍可少量参与派生防御，但不作为主要战斗数值来源。
- 默认优先做“字段语义正确 + 公式可解释 + 调试可观察”，而不是先做大量内容扩展。

## 完成定义

- 长期层：角色主属性正式从五维切换为六维
- 运行时：派生属性不再依赖旧 `精神` 与模糊 `先攻修正`
- 检定层：攻击、豁免、逃跑、恢复等公式切换到六维语义
- UI 层：测试页与状态栏能正确展示六维和关键派生
- 文档层：流程要求的说明文档全部同步完成

## 需要确认的一项取舍

- 推荐默认：`精神防御` 由 `感知` 主导，`治疗强度` 由 `感知` 主导，`MP 恢复/法术效率` 由 `智力` 主导。
- 若你更想要“更传统法师/牧师分家”的味道，则可把治疗强度改为 `感知 + 魅力` 或让部分神术技能走 `感知`、奥术技能走 `智力`。
- 这项选择会影响 `derive.ts`、`roll.ts`、`effects.ts` 和默认技能配置的分配方式。

### 当前确认结果

- 以“更符合 DND、尽量不容易出错、平衡性更稳”为优先目标，采用推荐默认分配：
  - `精神防御`：`感知` 主导
  - `治疗强度`：`感知` 主导
  - `MP 恢复/法术效率`：`智力` 主导
- 这样做的原因：
  - `感知` 承接意志、察觉、精神抗性与神术/支援稳定性，更接近 DND 语义
  - `智力` 承接奥术理解、法术效率、资源运用，边界清楚
  - 避免把 `魅力` 过早塞入治疗核心，降低平衡和解释成本
  - 有利于后续把速度体系稳定挂在 `敏捷` 上，而不再被旧 `精神/先攻修正` 牵扯
