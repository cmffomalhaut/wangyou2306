# 战斗技能引擎对齐检查清单

> 目的：确保 AI 按规则生成的技能可被战斗系统稳定导入、执行与回写。

## 1. 结构静态检查

### 1.1 必填字段检查

- 主动技能必须包含：
  - `类型`
  - `稀有度`
  - `元素属性`
  - `冷却回合`
  - `基础威力`
  - `描述`
  - `效果公式`
  - `目标类型`
  - `数值参数`
- 被动/天赋/光环必须包含：
  - `类型`
  - `稀有度`
  - `描述`
  - `效果公式`
  - `数值参数`

### 1.2 枚举合法性检查

- `类型` 仅允许：主动 / 被动 / 光环 / 天赋
- `稀有度` 仅允许：普通 / 稀有 / 史诗 / 传说 / 神话
- `元素属性` 仅允许：地 / 火 / 水 / 风 / 光 / 暗 / 无
- `目标类型` 仅允许：single_enemy / self / ally / all_enemies / all_allies
- `效果公式` 仅允许：
  - `physical_damage`
  - `magic_damage`
  - `heal`
  - `buff`
  - `debuff`
  - `drain_physical`
  - `drain_magic`

### 1.3 参数白名单检查

数值参数仅允许以下键：

- 伤害与命中：`系数`、`命中率`、`暴击率加成`、`吸血比例`
- 异常状态：`灼烧概率`、`灼烧伤害`、`灼烧回合`、`中毒概率`、`中毒伤害`、`中毒回合`、`冰冻概率`、`冰冻回合`、`麻痹概率`、`麻痹回合`、`混乱概率`、`混乱回合`
- Buff/Debuff：`攻击加成`、`防御加成`、`特攻加成`、`特防加成`、`速度加成`、`命中加成`、`闪避加成`、`持续回合`
- 治疗与护盾：`治疗量`、`治疗比例`、`护盾值`、`护盾比例`

### 1.4 数值区间检查

- 概率类字段：0~1
- `命中率`：0.35~1.0
- 比例类字段：0~1
- 回合类字段：正整数

---

## 2. 运行时行为检查

### 2.1 首技能铁律

- 第一技能必须是主动攻击技能
- 第一技能 `冷却回合=0`
- 第一技能 `基础威力>0`

### 2.2 目标语义对齐

- `single_enemy`：仅命中当前前排敌方
- `self/ally`：仅作用当前前排自身
- `all_allies + buff`：场地增益，仅前排并可换人继承
- `all_allies + heal`：可治疗全队（含后排）
- `all_enemies`：当前版本按前排结算，不应描述命中后排

### 2.3 公式语义一致性

- `效果公式` 与 `数值参数` 必须一致
- 禁止仅自然语言公式而无枚举
- 缺失 `系数` 时应允许引擎兜底而非报错

### 2.4 责任边界检查

- 战斗中 HP/MP/冷却变化由脚本结算
- AI 不在战斗中手算并回写上述运行时数值
- 战斗结束后由脚本统一回写战果

### 2.5 MP扣除一致性

- 技能导入时 `消耗MP` 可为0
- 实际扣蓝按稀有度固定值执行：普通5 / 稀有12 / 史诗20 / 传说32 / 神话45

---

## 3. 导入可用性检查

### 3.1 技能导入健壮性

- 任意战姬导入后，技能数组非空
- 目标类型无法识别时能落到默认行为，不导致战斗中断
- 无可用技能时可回退默认攻击

### 3.2 与引擎字段映射

对照以下实现点检查：

- `src/zhanji/新建为src文件夹中的文件夹/脚本/战斗系统/types.ts`
- `src/zhanji/新建为src文件夹中的文件夹/脚本/战斗系统/index.ts`
- `src/zhanji/新建为src文件夹中的文件夹/脚本/战斗系统/engine.ts`

### 3.3 回写安全性

- 战斗结束回写后，不破坏 `角色数据.*.技能.*` 原结构
- 不引入无法被后续回合识别的动态字段

---

## 4. 失败场景回归测试建议

- 测试“缺失系数”的主动技能：应可结算但伤害偏低
- 测试“目标类型缺失”的技能：应回退为引擎默认目标逻辑
- 测试“all_allies buff/治疗”差异：行为必须分离
- 测试“高概率文本但无数字”的技能：应在规则层被拦截，不进入战斗
- 测试“自定义未知参数名”：应在规则层判定不合规
