# 天香美人图 (Tavern Helper UI) - 整体设计详细方案

## 1. 设计概述

“天香美人图”是一件基于古典仙侠“合欢宗/法宝鼎炉”设定的超级法宝状态栏。在设计上，彻底抛弃了传统的扁平化控制面板，采用 **现代玻璃拟态 (Glassmorphism)** 与 **动态水墨卷轴 (Animated Scroll)** 结合的极致视觉体验，在满足极高美学要求的同时，为酒馆助手 (Tavern Helper) 前端界面提供完整的功能。

### 1.1 视觉风格 (Visual Aesthetic)
*   **主体材质**: 背景使用项目内置的高清宣纸纹理 (`paperboard-texture.png`)，结合 CSS `backdrop-filter` 实现了半透明毛笔纸和玻璃质感的融合。
*   **华丽卷轴**: 两侧卷轴柄 (`scroll-roller`) 经过深度 CSS 渲染。柄身为白玉质感并叠加 SVG 祥云/回字纹暗纹；轴帽为 3D 黄金质感，叠加了龙鳞/网格状的镂空 SVG 纹理。
*   **色情与权力的象征色**:
    *   **天香金** (`#d4af37` / `#ffe898`): 用于鎏金大字、边框和神圣法器感。
    *   **朱砂红/魅惑粉** (`#8a0b1c` / `#d12e53`): 用于表示情欲、血液、臣服与印记。
    *   **墨玄色** (`#1a1a1a`): 用于毛笔字迹。

## 2. 深度色情化命名系统 (The Erotic Nomenclature)

将常规的面板数值，进行了极度贴合“里世界修仙”语境的包装，营造出极强的统治感与沉浸感。

### 2.1 四大模块
*   **吾身**: 法宝主人的“真身本源”、“仙道五维”。
*   **幻卷**: 法宝自身的能量池与“天道法则演算”。
*   **芥子**: 内部小世界的控制中枢（包含日常校园、高压地下水牢等）。
*   **群芳**: 核心模块，更名为 **鼎炉名录**。

### 2.2 鼎炉名录 (卡片列表)
*   姓名 -> **真名** (象征被掌控)
*   当前所在 -> **肉身囚笼** (如：牝牝宫、地下水牢)
*   好感度 -> **倾慕 (依恋/敌意)**
*   臣服度 -> **牝犬印记 (发情/狂信)**
*   当前心声 -> **灵识谵语** (展示角色在屈辱与快感中的心理挣扎)

### 2.3 双修秘鉴 (详情页面)
*   外貌档案 -> **仙姿画骨**:
    *   胸部 -> **双峰秘景**
    *   阴部 -> **幽谷芳草**
    *   腿部 -> **玉腿缠腰**
    *   肛门 -> **后庭秘穴**
*   状态与性能力 -> **欲海沉沦**:
    *   性经历 -> **采补履历**
    *   性癖好 -> **淫根深种**
    *   性弱点 -> **破身死穴**
*   综合评价 -> **御女总纲**: 记录擒获手段、最佳推荐体位、最适宜的抽插节奏及双修禁忌。

## 3. 组件化拆分与工程结构

在 `src/yinyang/界面/状态栏/` 下，页面已被彻底重构为解耦的 Vue 3 SFC (Single File Components) 结构，便于酒馆助手在 `iframe` 中挂载与维护。

### `App.vue`
整个法宝的顶层容器。负责：
*   全屏环境遮罩与飘落的桃花粒子动画。
*   “祭出天香美人图”的初始召唤按钮。
*   调用 GSAP 统筹完整的“卷轴向两侧拉开 (3D 翻转 + ScaleX)”的入场与关闭动画。
*   管理四个主要模块 (Tab) 的状态切换。

### `components/NavSidebar.vue`
左侧导航侧边栏。
*   **残缺印泥设计**: 将常规的方形按钮利用 `clip-path` 裁剪出古老印章边缘的缺损感。
*   **物理手作感**: 通过 `:nth-child` 的 `even/odd` 为每枚印章加上了细微的正负旋转角度，模拟不完美的真实盖章痕迹。
*   悬浮与激活时，印章充血，爆发出强烈的朱砂红辉光。

### `components/LustBar.vue`
一个复用性极强的**液态波浪情欲条**组件。
*   弃用了传统的单色进度条，采用 SVG `<path>` 绘制波浪，通过 CSS `@keyframes` 让其在渐变色背景上不断滚动。
*   表现“灵力反哺”与“情欲泛滥”的流体视觉。

### `components/QunFang.vue` (鼎炉总成)
最庞大也是最核心的组件。内部实现了列表与详情 (`selectedGirl`) 两种状态的无缝 GSAP 动画切换。
*   列表态: 每一张“卡片”底部都垫着一枚半透明的莲花/淫纹阵法图案，Hover 时该图案会旋转放大并发出粉红魅魔光晕。
*   详情态: 利用 Grid 布局和大量的色情词汇构造出“不可描述的极品档案”。

### `components/WuShen.vue` & `components/HuanJuan.vue`
复用 `LustBar` 组件展示主人的基础面板和法宝的升级进度。目前内置了展示用的精美假数据，后续可通过 `getVariables({ type: 'character' })` 或 MVU 数据直接替换。

## 4. 后续接入指南 (Tavern Helper API Integration)

在酒馆脚本端实际连接数据时：
1. 请参考 `App.vue` 现有的生命周期，在 `onMounted` 后，通过酒馆助手自带的 `@types` 接口拉取真实数据。
2. 若采用了 MVU 变量库，可直接使用 `defineMvuDataStore(Schema)` 将本界面中的 `girls.value` 数据源与 `useDataStore().value.stat_data` 双向绑定。
3. 对外暴露的控制：当前是通过界面的 `isSummoned` 来控制法宝展开，在真实的 ST 脚本中，可以通过脚本上的【酒馆侧边栏扩展按钮】发送 Event 来修改该变量。
