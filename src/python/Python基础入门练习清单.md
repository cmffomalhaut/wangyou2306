# Python 基础入门练习清单

说明：这份清单与 `src/python/Python基础入门学习路径.md` 配套使用。目标不是做大量题，而是通过少量高频练习，把真正需要掌握的入门能力固定下来。

---

## 一、使用方式

- 每学完一个阶段，就做对应练习。
- 每个练习先自己完成，再对照运行结果检查。
- 卡住时优先描述：
  - 你想做什么
  - 你现在写了什么
  - 报错是什么
  - 你猜问题出在哪

---

## 二、阶段一练习：变量、输入输出、基本类型

### 练习 1：自我介绍脚本

要求：

- 输入姓名
- 输入年龄
- 输入城市
- 输出一句完整介绍

目标：

- 熟悉 `input()` 和 `print()`
- 熟悉字符串和变量

### 练习 2：基础计算器

要求：

- 输入两个数字
- 输出加、减、乘、除结果

目标：

- 熟悉类型转换
- 熟悉基本运算

### 练习 3：真假判断

要求：

- 输入一个数字
- 输出它是否大于 10
- 输出它是否是偶数

目标：

- 熟悉比较运算和布尔值

---

## 三、阶段二练习：条件与循环

### 练习 4：成绩评级

要求：

- 输入一个分数
- 按区间输出优秀、及格或不及格

目标：

- 熟悉 `if / elif / else`

### 练习 5：1 到 100 求和

要求：

- 分别用 `for` 和 `while` 写一遍

目标：

- 理解循环和计数变量

### 练习 6：猜数字简化版

要求：

- 预设一个数字
- 用户不断输入
- 猜对才结束
- 过大或过小时给提示

目标：

- 熟悉条件与循环组合

---

## 四、阶段三练习：字符串、列表、字典、集合

### 练习 7：文本清洗

要求：

- 输入一段文字
- 去掉首尾空格
- 全部转小写
- 把某个词替换成另一个词

目标：

- 熟悉字符串常用方法

### 练习 8：名单去重

要求：

- 给定一个姓名列表
- 去掉重复项
- 输出总人数

目标：

- 理解列表和集合的区别

### 练习 9：用户资料卡

要求：

- 用字典表示一个用户
- 包含姓名、年龄、职业、城市
- 格式化输出

目标：

- 熟悉字典的增删改查

### 练习 10：找最高分

要求：

- 用字典保存学生姓名和分数
- 输出最高分学生及其成绩

目标：

- 熟悉字典遍历

## 五、阶段四练习：函数

### 练习 11：判断偶数函数

要求：

目标：

- 熟悉参数与返回值

### 练习 12：列表去重函数

要求：

- 写一个函数，接收列表，返回去重后的新列表

目标：

- 练习封装重复逻辑

### 练习 13：成绩分析函数

要求：

- 写一个函数，接收学生分数字典
- 返回最高分学生姓名和分数

目标：

- 练习函数与字典组合使用

## 六、阶段五练习：文件与 JSON

- 熟悉文件读取

要求：

- 用户输入内容后尝试转成整数
- 输入非法时提示重新输入

目标：

- 熟悉 `try / except`

### 练习 19：修复列表越界

要求：

- 自己写一个会出现 `IndexError` 的小例子
- 再修复它

- 理解索引边界

### 练习 20：修复文件不存在问题

要求：

## 八、阶段七练习：最小项目

### 项目 1：文本统计工具

最低要求：

- 输入文件路径
- 输出行数
- 输出字符数
- 输出指定关键词出现次数

### 项目 2：通讯录查询工具

最低要求：

- 用字典保存若干联系人
- 支持按姓名查询电话
- 查询不到时给出提示

### 项目 3：待办事项记录器

最低要求：

- 支持新增事项
- 支持查看事项列表
- 支持保存到 JSON 文件

### 项目 4：公开 API 查询脚本

最低要求：

- 安装一个请求库
- 调用公开接口
- 解析 JSON
- 打印一到两个关键字段

## 九、自检标准

如果下面大部分都能独立完成，说明入门已经站稳：

- 能写简单输入输出脚本
- 能写条件判断和循环
- 能使用列表、字典和字符串
- 能把逻辑拆成函数
- 能读写文本和 JSON
- 能安装第三方库
- 能看懂常见 traceback
- 能做出 2 到 4 个小工具

---

## 十、练习时最容易犯的错误

- 忘记把 `input()` 结果转成数字
- 把 `=` 和 `==` 混淆
- 列表索引越界
- 字典键名写错
- 缩进不一致
- 只会 `print()` 结果，不会 `return` 结果
- 文件路径写错
- 报错后直接乱改，而不是先看 traceback

---

## 十一、练习阶段的正确节奏

- 每次只做一个小练习，不要堆太多。
- 每完成一个练习，就运行并验证。
- 遇到错误先自己判断类型，再寻求 AI 帮助。
- 把重点放在理解和复盘，不放在刷数量。

完成这份清单后，再进入 `requests`、`pandas`、SQL 或 Web 开发会更稳。
