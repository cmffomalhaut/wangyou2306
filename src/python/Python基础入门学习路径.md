# Python 基础入门学习路径

说明：本文档是从计划稿落地后的正式学习文档，目标不是把 Python 全部讲完，而是整理出一条适合零基础、偏实用、偏 AI 协作场景的入门主线。完成这份路径后，应该能够看懂常见 Python 脚本、修改简单程序、处理基础报错，并写出几个能实际解决问题的小工具。

---

## 一、学习目标

这份路径只要求达到入门可用，不追求一开始就掌握完整工程能力。

完成后应达到的水平：

- 能运行 Python 脚本，知道代码是怎样被执行的。
- 能看懂常见基础语法和简单控制流程。
- 能使用列表、字典、字符串处理基础数据。
- 能写函数，把重复逻辑拆分开。
- 能读写文本文件和 JSON 文件。
- 能安装第三方库，理解虚拟环境的用途。
- 能看懂常见 traceback，处理基础报错。
- 能写出几个小脚本，并配合 AI 继续扩展。

---

## 二、学习边界

### 当前必须学的

- 基础语法
- 常见数据类型

- 函数
- 模块与导入
- 文件读写

- 异常与调试
- `pip` 和虚拟环境的基础概念

### 当前不用深挖的

## 三、推荐学习顺序总览

1. 先能运行 Python，理解变量、输入输出、基本类型。
2. 学会 `if`、`for`、`while`，理解代码执行流程。
3. 掌握字符串、列表、字典、集合。
4. 学会写函数，理解参数和返回值。
5. 学会 `import`、文件读写、JSON 和路径。
6. 学会安装库、认识虚拟环境。
7. 学会看 traceback、处理异常和做基础调试。
8. 做几个最小实用脚本，把前面内容串起来。

## 四、第一阶段：认识 Python 与最小程序结构

### 阶段目标

- 能安装并运行 Python。
- 能理解最简单脚本的执行顺序。
- 能区分变量、值和表达式。

### 必学内容

#### 1. Python 是什么

- Python 是一种通用编程语言。
- 常用于自动化、脚本、数据处理、后端、AI 工具链。
- 你写的代码通常保存在 `.py` 文件中。
- 运行方式通常是：

```bash
python your_script.py
```

#### 2. 程序是顺序执行的

```python
print("第一行")
print("第二行")
print("第三行")
```

- 默认情况下，程序按从上到下的顺序执行。
- 这是理解后面条件、循环、函数的基础。

#### 3. 输入与输出

```python
name = input("请输入你的名字: ")
print("你好", name)
```

- `input()` 读取输入，结果默认是字符串。
- `print()` 把结果输出到终端。

#### 4. 变量与赋值

```python
age = 18
city = "Shanghai"
```

- 变量是“名字绑定到一个值”。
- `=` 在 Python 里是赋值，不是数学推导。
- 变量名要表达含义，避免长期使用无意义名称。

#### 5. 基本数据类型

必须认识这几个类型：

- `int`：整数
- `float`：小数
- `str`：字符串
- `bool`：布尔值，只有 `True` 和 `False`
- `None`：空值、缺省值

示例：

```python
age = 18
price = 9.9
name = "Alice"
is_student = True
result = None
```

#### 6. 基本运算

- 算术运算：`+ - * / // % **`
- 比较运算：`== != > < >= <=`
- 逻辑运算：`and or not`

重点理解：

- `==` 是比较是否相等。
- `=` 是赋值。
- `/` 是普通除法。
- `//` 是整除。
- `%` 是取余。

### 这一阶段要会的事

- 运行一个最简单的 `.py` 文件。
- 从键盘读取输入并输出结果。
- 完成简单数值计算。
- 看懂变量赋值和基本表达式。

### 建议练习

- 输入姓名和年龄，输出一句自我介绍。
- 输入两个数字，输出四则运算结果。
- 输入一个数字，输出它是否大于 10。

## 五、第二阶段：学会控制程序流程

### 阶段目标

- 理解程序如何根据条件走不同路径。
- 理解如何重复执行某一段逻辑。

### 必学内容

#### 1. 条件判断 `if / elif / else`

```python
score = 85

if score >= 90:
    print("优秀")
elif score >= 60:
    print("及格")
else:
    print("不及格")
```

- `if` 表示“如果满足条件”。
- `elif` 表示“否则如果”。
- `else` 表示“前面的条件都不满足时”。

#### 2. 缩进就是语法

- Python 用缩进表示代码块的归属关系。
- 缩进错了，逻辑就会变，甚至直接报错。
- 初学阶段要养成习惯：看到缩进就判断这行代码属于哪个结构。

#### 3. `for` 循环

```python
for item in [1, 2, 3]:
    print(item)
```

- `for` 用于遍历一组数据。
- 后面处理列表、字符串、字典时都会大量使用。

#### 4. `range()`

```python
for i in range(5):
    print(i)
```

- `range(5)` 会生成 `0, 1, 2, 3, 4`。
- 常用于固定次数循环。

#### 5. `while` 循环

```python
count = 0
while count < 3:
    print(count)
    count += 1
```

- 当条件为真时一直执行。
- 需要特别小心死循环。

#### 6. `break` 和 `continue`

- `break`：提前结束整个循环。
- `continue`：跳过当前这一次，继续下一轮。

### 这一阶段要会的事

- 根据条件输出不同结果。
- 用 `for` 和 `while` 完成简单重复任务。
- 理解缩进造成的执行范围差异。

### 建议练习

- 输入分数，输出评级。
- 输出 1 到 10。
- 计算 1 到 100 的和。
- 模拟密码输入，输对才结束。

## 六、第三阶段：掌握最常用的数据结构

### 阶段目标

- 开始处理成组数据，而不是只处理单个值。
- 能看懂常见的数据组织方式。

### 必学内容

#### 1. 字符串 `str`

- 文本就是字符串。
- 入门要会：
  - 拼接
  - `f-string`
  - `strip()`
  - `lower()` / `upper()`
  - `split()`
  - `replace()`

```python
name = "Alice"
print(f"Hello, {name}")
```

#### 2. 列表 `list`

```python
numbers = [10, 20, 30]
```

- 列表是有顺序的容器。
- 必须会：
  - 索引取值：`numbers[0]`
  - 切片：`numbers[1:3]`
  - `append()`
  - `remove()`
  - `pop()`
  - `sort()`
  - `len()`

重点理解：

- 索引从 `0` 开始。
- 列表是可变的。

#### 3. 字典 `dict`

```python
user = {"name": "Alice", "age": 18}
```

- 字典是键值对结构。
- 非常适合表示一条记录、一份配置、一项数据对象。

必须会：

- 取值：`user["name"]`
- 安全取值：`user.get("name")`
- 新增和修改键值
- `items()` 遍历

#### 4. 集合 `set`

```python
tags = {"python", "ai", "python"}
```

- 集合最常见用途是去重。
- 也可以快速判断某元素是否存在。

#### 5. 常见遍历方式

```python
for item in items:
    print(item)

for key, value in user.items():
    print(key, value)
```

### 这一阶段要会的事

- 用列表保存多条数据。
- 用字典表示结构化数据。
- 对字符串做拆分、替换、格式化。
- 读懂列表嵌套字典这类高频结构。

### 建议练习

- 去重一个姓名列表。
- 统计一句话里某个词出现的次数。
- 用字典保存用户资料并打印出来。
- 遍历学生分数字典，找出最高分。

## 七、第四阶段：函数与代码复用

### 阶段目标

- 学会把一段逻辑封装起来。
- 理解为什么代码不能一直堆在全局区域。

### 必学内容

#### 1. 定义函数

```python
def greet(name):
    print(f"Hello, {name}")
```

- 函数是一段可以重复调用的逻辑。

#### 2. 调用函数

```python
greet("Alice")
```

- 定义不等于执行。
- 只有调用时才会真正运行。

#### 3. 参数与返回值

```python
def add(a, b):
    return a + b
```

重点理解：

- 参数是输入。
- `return` 是把结果交还给调用方。
- `print()` 给人看，`return` 给程序后续使用。

#### 4. 默认参数

```python
def greet(name, message="Hello"):
    print(message, name)
```

- 只需要掌握基本写法。
- 暂时不深入高级参数规则。

#### 5. 作用域的最小概念

- 函数内部的变量通常只在函数内部有效。
- 入门阶段尽量少依赖全局变量。

#### 6. 拆分主流程

初学时应逐渐养成这种意识：

- 一个函数负责输入处理。
- 一个函数负责计算或整理数据。
- 一个函数负责输出结果。

### 这一阶段要会的事

- 把重复逻辑封装成函数。
- 用参数传入数据，用返回值拿到结果。
- 看懂一份简单脚本中的函数调用关系。

### 建议练习

- 写一个函数判断偶数。
- 写一个函数给列表去重。
- 写一个函数求字典中最高分学生。

## 八、第五阶段：模块、文件与 JSON

### 阶段目标

- 从“只会写控制台小片段”进入“能处理文件和多文件脚本”。

### 必学内容

#### 1. 模块与导入

```python
import math
from pathlib import Path
```

- 一个 `.py` 文件就可以是一个模块。
- `import` 的本质是把别处的代码或工具拿来使用。

#### 2. `if __name__ == "__main__":`

```python
def main():
    print("run")

if __name__ == "__main__":
    main()
```

- 入门阶段只要理解它的作用：
  - 让这个文件既能被直接运行，也能被其他文件导入。

#### 3. 文件读写

```python
with open("data.txt", "r", encoding="utf-8") as f:
    content = f.read()
```

- 必须掌握：
  - 文本读取
  - 文本写入
  - `with open(...)` 的基本结构

重点理解：

- 中文文件常常要显式写 `encoding="utf-8"`。
- 读取模式和写入模式不同。

#### 4. 路径概念

- 要知道相对路径和绝对路径的区别。
- 推荐开始使用 `pathlib.Path` 处理路径。

#### 5. JSON 处理

```python
import json

with open("data.json", "r", encoding="utf-8") as f:
    data = json.load(f)
```

- 必须掌握：
  - `json.load()` / `json.loads()`
  - `json.dump()` / `json.dumps()`

重点理解：

- JSON 通常就是列表和字典的嵌套。
- 现实开发里它极其常见。

### 这一阶段要会的事

- 读一个文本文件。
- 写一个文本文件。
- 读写 JSON。
- 把脚本拆成两个文件并导入使用。

### 建议练习

- 统计一个文本文件的行数和单词数。
- 读取一个 JSON 文件，提取指定字段。
- 把一组数据保存成 JSON 文件。
- 把一段逻辑拆到独立模块中。

## 九、第六阶段：安装库与基础工程常识

### 阶段目标

- 理解“自己的代码”和“别人写好的库”如何配合。
- 理解为什么不同项目要有隔离环境。

### 必学内容

#### 1. `pip` 是什么

- `pip` 是 Python 常用包管理工具。
- 用来安装第三方库。

示例：

```bash
pip install requests
```

#### 2. 标准库与第三方库的区别

- 标准库：Python 自带，不需要额外安装。
- 第三方库：别人发布的扩展，需要安装。

#### 3. 虚拟环境的最小概念

- 虚拟环境用于隔离项目依赖。
- 不同项目可以有不同版本的库，避免互相干扰。

需要理解的不是所有命令细节，而是它解决的问题：

- 防止全局环境越来越乱。
- 让项目依赖更可控。

#### 4. 依赖是项目的一部分

- 一段代码能否运行，不只看源码，还看依赖是否齐全。
- 以后你读别人的项目时，要形成“先看依赖”的意识。

### 这一阶段要会的事

- 安装一个常用第三方库。
- 知道为什么有的代码本机能跑，换一台机器就不行。
- 理解虚拟环境存在的理由。

### 建议练习

- 安装 `requests` 并导入它。
- 安装一个小工具库并尝试读取其官方示例。
- 用自己的话解释什么是虚拟环境。

## 十、第七阶段：异常、traceback 与调试

### 阶段目标

- 面对报错不慌。
- 能定位大多数基础问题的大致原因。

### 必学内容

#### 1. 学会看 traceback

- 先看最后一行异常类型和报错信息。
- 再看出错文件和行号。
- 再回到对应变量、索引、键名或输入数据。

#### 2. 必须认识的常见异常

- `SyntaxError`
- `NameError`
- `TypeError`
- `ValueError`
- `IndexError`
- `KeyError`
- `FileNotFoundError`

你不需要背定义，但要知道这些错误大概意味着什么。

#### 3. `try / except`

```python
try:
    value = int(input("请输入数字: "))
except ValueError:
    print("输入不是合法数字")
```

重点理解：

- `try` 是对可能失败操作的保护。
- `except` 负责有针对性地处理错误。
- 不要为了“别报错”就乱写空的 `except`。

#### 4. 最基础的调试方法

- 用 `print()` 查看变量值。
- 把复杂问题缩小为最小可复现案例。
- 分清是语法错误、运行错误还是逻辑错误。

#### 5. 三类错误的区别

- 语法错误：代码还没开始正常运行就失败。
- 运行错误：运行中遇到非法操作而崩溃。
- 逻辑错误：不报错，但结果不对。

### 这一阶段要会的事

- 根据 traceback 找到错误大致位置。
- 自己修掉基础变量名错误、索引错误、类型错误。
- 给用户输入加上基础校验。

### 建议练习

- 故意写出几种常见异常并解释原因。
- 写一个安全读取数字输入的小程序。
- 修复一个字典缺少键、列表越界、文件路径错误的例子。

## 十二、入门阶段最重要的习惯

### 1. 写清晰代码，不写炫技代码

- 入门阶段最优先的是可读性。
- 看得懂、改得动，比一行写完更重要。

### 2. 变量命名要表达含义

- `user_name` 通常比 `a` 更好。
- 清晰命名会直接提高调试效率。

### 3. 一个函数尽量只做一件事

- 更容易测试。
- 更容易修改。
- 更适合和 AI 协作。

### 4. 遇到错误先定位，不要乱改一大片

- 先看报错位置。
- 再找相关变量。
- 最后再修改。

### 5. 保持“输入 -> 处理 -> 输出”的结构意识

- 大多数入门脚本都可以这样理解。
- 有了这个框架，阅读别人代码会轻松很多。

## 十四、如何配合 AI 使用这份路径

建议做法：

- 每次只学一节，不要试图一口气读完。
- 每学完一节，就做 1 到 3 个小练习。
- 练习遇到问题时，把以下信息一起给 AI：
  - 你的代码
  - 报错信息
  - 你预期的结果
  - 实际发生的结果
- 不要只说“这段代码不对”，而要尽量描述上下文。

对 AI 最有效的提问方式通常是：

- “解释这段代码每一行在做什么。”
- “这段 traceback 的关键报错是什么。”
- “请不要重写全部代码，只指出我哪里错了。”
- “用初学者能理解的方式解释这段函数。”
